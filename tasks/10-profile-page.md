# 10. Profile Page

## Цель
Личный кабинет пользователя: профиль, безопасность, подписка, прогресс.

## Маршрут
`/profile`

## Структура (Tabs)

1. **Профиль** — ФИО, email (read-only), аватар (загрузка через Cloudinary, см. `08-creator-studio.md`).
2. **Безопасность** — смена пароля (current + new + confirm).
3. **Подписка** — текущий статус, план, сроки (`startsAt`/`expiresAt`), история подписок (компактная таблица).
4. **Прогресс** — список курсов, по которым есть прогресс, с прогресс-баром и кнопкой «Продолжить».

## Запросы
- `GET /api/v1/users/me`
- `PATCH /api/v1/users/me`
- `PATCH /api/v1/users/me/password`
- `GET /api/v1/me/subscription`
- `GET /api/v1/me/subscriptions`
- `GET /api/v1/me/progress`

## Компоненты
- `<ProfileForm />`
- `<PasswordForm />`
- `<SubscriptionCard />`
- `<ProgressList />`
- `<AvatarUploader />`

## UX
- Сохранение профиля — оптимистичное обновление + toast.
- При смене пароля бэк может разлогинить — фронт принудительно делает `logout` и редиректит на `/auth/login`.
- Если подписки нет — `<EmptyState />` с текстом «Обратитесь к администратору для получения доступа».

## Definition of Done
- [ ] Все 4 таба работают.
- [ ] Аватар грузится и сразу отображается.
- [ ] Смена пароля валидируется (zod + сравнение с current).
- [ ] Прогресс-бары корректно показывают %.
