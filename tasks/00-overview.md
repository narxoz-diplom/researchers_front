# 00. Общий обзор продукта

> Этот файл идентичен файлу `researchers_back/tasks/00-overview.md` и служит единым источником правды о продукте для команд бэкенда и фронтенда.

## Идея

**researchers** — учебная платформа с ручным наполнением. Авторы загружают курсы, состоящие из уроков (текст + видео + файлы), а подписчики потребляют контент после выдачи им подписки админом. **Никаких ИИ-функций, генераций или RAG.**

## Целевые пользователи

- **Author** — преподаватель/эксперт, который вручную создаёт курсы.
- **Subscriber** — учащийся, потребляет контент после получения подписки.
- **Admin** — оператор платформы, управляет ролями и подписками.

## Ключевые фичи (MVP)

1. **Авторизация** — регистрация / вход / выход / refresh токена.
2. **Каталог курсов** — публичный список опубликованных курсов.
3. **Страница курса** — описание, обложка, список уроков, бейдж «Требуется подписка».
4. **Просмотр урока** — видеоплеер + контент урока + скачивание прикреплённых файлов.
5. **Creator Studio** — для автора: CRUD курсов и уроков, загрузка видео и файлов.
6. **Admin Panel** — для админа: смена ролей, выдача/отзыв подписок.
7. **Личный профиль** — данные пользователя, статус подписки.
8. **Прогресс** — отметка пройденных уроков.

## Что НЕ входит в MVP

- AI/RAG/чат-боты внутри уроков.
- Тесты, экзамены, автопроверки.
- Платёжная интеграция (Stripe, Kaspi и пр.) — подписка выдаётся вручную.
- Уведомления по email/push.
- Комментарии под уроками.
- Многоязычные версии курсов.
- Мобильное приложение.

## Бизнес-правила

- Регистрация по умолчанию создаёт пользователя с ролью `SUBSCRIBER`.
- Только `ADMIN` может менять роли других пользователей.
- Только автор курса (или `ADMIN`) может редактировать/удалять курс.
- Курс становится виден подписчикам только после статуса `PUBLISHED`.
- `SUBSCRIBER` получает доступ к содержимому уроков (видео + материалы) только при наличии активной подписки.
- Подписка имеет срок (`expiresAt`); по истечении переходит в `EXPIRED`.
- `ADMIN` и `AUTHOR` (для своих курсов) могут смотреть содержимое без подписки.

## Стек

- **Backend:** NestJS + PostgreSQL + Prisma + JWT + Cloudinary SDK.
- **Frontend:** React + Vite + TypeScript + TanStack Query + Tailwind + shadcn/ui.
- **Хранилище медиа:** Cloudinary (через signed uploads).

## Архитектурный контракт между фронтом и бэком

- REST API под `/api/v1/*`, JSON.
- Авторизация — Bearer JWT в заголовке `Authorization`.
- Видео и файлы фронт берёт **напрямую с Cloudinary** по `secure_url`, который отдаёт бэк.
- Загрузка нового видео/файла идёт **напрямую с фронта на Cloudinary** через подпись, выданную бэком (`POST /api/v1/media/sign`), затем фронт отправляет на бэк только `public_id` + метаданные.
