# 07. Lesson Player Page

## Цель
Двухколоночный экран урока: видео + контент + материалы + список уроков справа.

## Маршрут
`/courses/:courseId/lessons/:lessonId`

## Источник данных
- `GET /api/v1/lessons/:id` → детали урока с видео и материалами (`403 SUBSCRIPTION_REQUIRED` если нет доступа).
- `GET /api/v1/courses/:courseId/lessons` — для сайдбара навигации.
- `POST /api/v1/lessons/:id/complete` — отметить пройденным.

## Макет

```
+--------------------------------------+--------------------------+
| <BackToCourse /> Заголовок урока     |  Уроки курса             |
| +----------------------------------+ |  ▸ 1. Введение  ✓        |
| | <VideoPlayer />                  | |  ▸ 2. Тема 1    ●(active)|
| +----------------------------------+ |  ▸ 3. Тема 2             |
|                                      |  ...                     |
| Текст урока (markdown)              |                          |
|                                      |                          |
| Материалы:                          |                          |
|  - Slides.pdf  [Скачать]            |                          |
|  - Code.zip    [Скачать]            |                          |
|                                      |                          |
| [ ◀ Предыдущий ]   [ Следующий ▶ ]  |                          |
| [ Отметить как пройденный ]         |                          |
+--------------------------------------+--------------------------+
```

## Компоненты

- `<VideoPlayer src={url} poster={...} />` — обёртка над `react-player` с кастомным контролом, поддержкой клавиатуры (`Space`, `←`, `→`, `F`).
- `<LessonSidebar lessons currentId progressMap />` — список с состояниями (locked/active/completed).
- `<MaterialList materials />` — каждый материал имеет иконку по `mimeType`, размер, кнопку «Скачать» (просто `<a href={url} download>`).
- `<MarkdownContent content />` — рендер контента урока (react-markdown + sanitize).

## Поведение

- При `403 SUBSCRIPTION_REQUIRED` → отрисовать `<PaywallState />` с CTA.
- «Отметить как пройденный» — оптимистичный апдейт, после успеха обновляет sidebar.
- Кнопка «Следующий» автоматически появляется, если есть следующий урок.

## Definition of Done
- [ ] Видео стримится с Cloudinary, поддерживает паузу / перемотку.
- [ ] Материалы скачиваются.
- [ ] Mark-as-completed работает оптимистично и синхронизирует прогресс.
- [ ] При отсутствии подписки рендерится `<PaywallState />`.
- [ ] Адаптив: на мобильном сайдбар сворачивается в `<Sheet />`.
